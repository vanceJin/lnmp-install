# LNMP 安装脚本 - 快速部署指南

## 方法 1：使用 base64 编码传输（推荐，适合大文件）

### 在 Windows PowerShell 中执行：

```powershell
# 1. 将脚本编码为 base64
$base64 = [Convert]::ToBase64String([IO.File]::ReadAllBytes("c:\Users\work_wang\Documents\trae_projects\docker\lnmp\install-lnmp.sh"))

# 2. 通过 SSH 传输并解码（替换为你的服务器信息）
$base64 | ssh root@你的服务器IP "base64 -d > /root/install-lnmp.sh && chmod +x /root/install-lnmp.sh"
```

### 示例（替换 IP 和用户名）：
```powershell
$base64 = [Convert]::ToBase64String([IO.File]::ReadAllBytes("c:\Users\work_wang\Documents\trae_projects\docker\lnmp\install-lnmp.sh"))
$base64 | ssh root@192.168.1.100 "base64 -d > /root/install-lnmp.sh && chmod +x /root/install-lnmp.sh"
```

## 方法 2：使用 scp（最简单）

```powershell
# 直接复制文件到服务器
scp c:\Users\work_wang\Documents\trae_projects\docker\lnmp\install-lnmp.sh root@你的服务器IP:/root/

# 然后登录服务器设置执行权限
ssh root@你的服务器IP "chmod +x /root/install-lnmp.sh"
```

---

## 方法 3：使用 cat 和 heredoc（适合小文件）

由于你的脚本太大（800+ 行），不建议使用 heredoc 方式，容易出错。

如果一定要用，可以分段传输：

```bash
# 先传输第一部分
ssh root@你的服务器IP "cat > /root/install-lnmp.sh << 'EOF'
#!/bin/bash
# ... 前 200 行内容
EOF"

# 再追加其他部分
ssh root@你的服务器IP "cat >> /root/install-lnmp.sh << 'EOF'
# ... 接下来的内容
EOF"
```

---

## 方法 4：使用 curl 或 wget（如果服务器能访问外网）

### 方案 A：上传到 GitHub Gist

1. 访问：https://gist.github.com
2. 粘贴脚本内容
3. 创建后获取 raw 链接
4. 在服务器上执行：

```bash
wget -O /root/install-lnmp.sh https://gist.githubusercontent.com/你的用户名/xxx/raw/install-lnmp.sh
chmod +x /root/install-lnmp.sh
```

### 方案 B：上传到自己的服务器

如果你有可访问的 Web 服务器：

```bash
# 在服务器上下载
wget -O /root/install-lnmp.sh http://你的域名/install-lnmp.sh
chmod +x /root/install-lnmp.sh
```

## 推荐使用方法

**最简单的方式（方法 2）：**

```powershell
# 1. 上传文件
scp c:\Users\work_wang\Documents\trae_projects\docker\lnmp\install-lnmp.sh root@你的服务器IP:/root/

# 2. 登录并设置权限
ssh root@你的服务器IP

# 3. 设置执行权限
chmod +x /root/install-lnmp.sh

# 4. 运行安装
cd /root
./install-lnmp.sh
```

## 快速测试

上传完成后，在服务器上执行：

```bash
# 检查文件是否存在
ls -lh /root/install-lnmp.sh

# 查看文件前 10 行
head -n 10 /root/install-lnmp.sh

# 运行安装
sudo ./install-lnmp.sh
```

---

## 如果 SSH 连接失败

检查以下几点：
1. 服务器 IP 是否正确
2. SSH 端口（默认 22）是否开放
3. 防火墙是否允许 SSH 连接
4. 密码或 SSH 密钥是否正确

可以使用以下命令测试连接：
```powershell
ssh -v root@你的服务器IP
```
